# Rubric Packs

Use one pack per decision run, then tailor names/weights as needed. Keep total weight at `100`.

## Pack A: Universal Default

Best for broad comparisons (skills, workflows, tools, architecture options).

| Criterion                             | Weight |
| ------------------------------------- | -----: |
| Use-case coverage                     |     25 |
| Workflow rigor and repeatability      |     20 |
| Objective measurability               |     15 |
| Trigger precision and discoverability |     10 |
| Platform portability/interoperability |     10 |
| Maintainability and extensibility     |     10 |
| Adoption cost / time-to-value         |     10 |

## Pack B: Tool or Vendor Selection

Best for product evaluations and buy-side decisions.

| Criterion                      | Weight |
| ------------------------------ | -----: |
| Capability fit                 |     25 |
| Integration complexity         |     20 |
| Reliability and operability    |     15 |
| Total cost of ownership        |     15 |
| Security and compliance risk   |     10 |
| Portability / lock-in exposure |     10 |
| Adoption speed                 |      5 |

## Pack C: Architecture Decision

Best for platform and system design alternatives.

| Criterion                        | Weight |
| -------------------------------- | -----: |
| Functional fit                   |     20 |
| Scalability and performance      |     20 |
| Reliability and resilience       |     15 |
| Operability and observability    |     15 |
| Security and compliance          |     10 |
| Maintainability and evolvability |     10 |
| Delivery effort and timeline     |     10 |

## Pack D: Build-vs-Buy

Best for in-house implementation versus commercial solution.

| Criterion                       | Weight |
| ------------------------------- | -----: |
| Strategic fit and control       |     20 |
| Time-to-value                   |     20 |
| Total cost horizon              |     20 |
| Risk/compliance/vendor exposure |     15 |
| Integration complexity          |     10 |
| Customization flexibility       |     10 |
| Ongoing maintenance burden      |      5 |

## Calibration Notes

- Use `1-5` when evidence is mostly qualitative and workshop-driven.
- Use `0-100` when evidence includes measurable benchmarks.
- Keep the same rubric, scale, and platforms for all alternatives in one run.
